# 任务池（MissionPools）功能文档

---

## 概述

任务池是一个集中式任务管理系统，用于调度和管理各种类型的游戏任务。每个房间维护独立的任务池，Creep 从任务池中获取任务并执行，完成后提交完成信息。

---

## 核心概念

### 1. 任务池（MissionPools）

每个房间都有独立的任务池，存储在 `Memory.RosmarinBot.MissionPools[roomName]` 中。

### 2. 任务（Task）

任务是任务池中的最小单元，包含任务类型、优先级、数据和状态。

**任务状态：**
- `pending`：待处理
- `locked`：已锁定，其他 Creep 无法获取
- `completed`：已完成

---

## 数据结构

### 任务池结构

```typescript
interface MissionPoolMemory {
    [missionType: string]: Task[]  // 任务类型 -> 任务数组
}
```

### 任务结构

```typescript
interface Task {
    id: string;                    // 任务唯一ID，格式如 "transport-XXXX"
    type: string;                  // 任务类型
    level: number;                 // 优先级，数值越小优先级越高（0为最高）
    data: any;                     // 任务数据，根据类型不同而不同
    lock?: boolean;                // 是否被锁定
    bindCreep?: Id<Creep> | null;  // 绑定该任务的 Creep ID
}
```

## 任务类型

| 类型 | 说明 | 数据结构 |
|------|------|----------|
| `transport` | 运输任务 | `{source, target, resourceType, amount, pos}` |
| `manage` | 中央搬运任务 | `{source, target, resourceType, amount}` |
| `build` | 建造任务 | `{target, pos}` |
| `repair` | 维修任务 | `{target, pos, hits}` |
| `send` | 发送任务 | `{targetRoom, resourceType, amount}` |
| `spawn` | 孵化任务 | `{name, body, memory, energy, upbody}` |
| `mine` | 采矿任务 | `{target, sourceType, roomName}` |
| `boost` | 强化任务 | `{mineral, totalAmount, owners}` |

## 任务生命周期

### 1. 任务生成

各模块根据游戏状态生成任务：

```typescript
// 添加运输任务
room.TransportMissionAdd(level, {
    pos: compress(pos.x, pos.y),
    source: sourceId,
    target: targetId,
    resourceType: RESOURCE_ENERGY,
    amount: 1000
});

// 添加孵化任务
room.SpawnMissionAdd(name, body, level, role, memory);

// 添加发送任务
room.SendMissionUpsertMax(targetRoom, resourceType, amount, maxAmount);
```

### 2. 任务获取

Creep 从任务池获取任务：

```typescript
// 获取运输任务（考虑距离和优先级）
const task = room.getTransportMission(creep);

// 获取孵化任务（考虑能量是否足够）
const task = room.getSpawnMission();

// 获取建造任务
const task = room.getBuildMission(creep);
```

**任务分配策略：**
1. **优先级**：level 数值越小优先级越高
2. **距离优化**：同优先级任务按切比雪夫距离分配
3. **锁定检查**：已锁定的任务不会被分配

### 3. 任务锁定

任务被分配给特定 Creep 后锁定：

```typescript
room.lockMissionInPool('transport', task.id, creep.id);
```

锁定后：
- `task.lock = true`
- `task.bindCreep = creep.id`
- 其他 Creep 无法获取该任务

### 4. 任务执行

Creep 执行任务逻辑（根据任务类型不同）：

```typescript
// 运输任务执行（transport 角色在 transport.ts 中处理）
const transport = creep.memory.mission.data as TransportTask;

// 从源建筑取资源
if (creep.pos.isNearTo(sourceObj)) {
    creep.withdraw(sourceObj, resourceType, amount);
}
// 向目标建筑存资源
else if (creep.pos.isNearTo(targetObj)) {
    const transferAmount = Math.min(amount, creep.store[resourceType], targetObj.store.getFreeCapacity(resourceType));
    creep.transfer(targetObj, resourceType, transferAmount);
}
```

### 5. 任务提交

任务完成后提交完成信息：

```typescript
// 提交运输任务（减少剩余数量）
room.submitTransportMission(task.id, 500);

// 提交孵化任务（直接删除）
room.submitSpawnMission(task.id);

// 提交建造任务
room.submitBuildMission(task.id);
```

### 6. 任务清理

系统自动清理已完成或失效的任务：

```typescript
// 检查并清理运输任务池
room.checkMissionPool('transport', (task) => {
    return task.data.amount > 0;  // 只有剩余数量>0的任务才保留
});

// 检查并清理建造任务池
room.checkMissionPool('build', (task) => {
    const target = Game.getObjectById(task.data.target);
    return target !== null;  // 只有工地仍存在的任务才保留
});
```

## 任务优先级

### 优先级规则

- 数值越小，优先级越高
- 0 为最高优先级
- 通常范围：0-10

### 常用优先级

| 任务类型 | 优先级 | 说明 |
|----------|--------|------|
| `spawn` | 0-2 | 最高优先级，确保孵化不阻塞 |
| `repair` | 1-3 | 高优先级，关键设施维修 |
| `boost` | 1-3 | 高优先级，Team 需求 |
| `transport` | 3-6 | 中等优先级，资源搬运 |
| `manage` | 4-7 | 中等优先级，资源调配 |
| `build` | 5-8 | 较低优先级，非紧急建造 |
| `mine` | 6-9 | 低优先级，PowerBank/Deposit |

## 任务调度

### 分帧调度

为降低 CPU 峰值，任务更新按不同频率执行：

```typescript
const schedule = [
    { interval: 10, offset: 0, run: (offset) => this.UpdateSpawnMission(offset) },
    { interval: 20, offset: 1, run: (offset) => this.UpdateTransportMission(offset) },
    { interval: 20, offset: 2, run: (offset) => this.UpdateBoostMission(offset) },
    { interval: 30, offset: 3, run: (offset) => this.UpdateManageMission(offset) },
    { interval: 30, offset: 4, run: (offset) => this.UpdateBuildMission(offset) },
    { interval: 30, offset: 5, run: (offset) => this.UpdateRepairMission(offset) },
];
```

### 调度策略

- **偏移执行**：不同任务类型在不同 tick 执行，分散 CPU 开销
- **间隔控制**：高优先级任务更新频率更高
- **偏移参数**：相同任务类型的房间错开执行时间

## 距离优化

### 切比雪夫距离

任务分配使用切比雪夫距离（Chebyshev Distance）：

```typescript
const chebyshevDistance = (pos1: RoomPosition, pos2: RoomPosition): number => {
    return Math.max(Math.abs(pos1.x - pos2.x), Math.abs(pos1.y - pos2.y));
};
```

### 距离权重

同优先级任务按距离分配，优先分配给距离最近的 Creep：

```typescript
const getBestTask = (creep: Creep, tasks: Task[]): Task => {
    let bestTask: Task | null = null;
    let bestDistance = Infinity;

    for (const task of tasks) {
        if (task.lock) continue;  // 跳过已锁定任务
        const distance = chebyshevDistance(creep.pos, task.pos);
        if (distance < bestDistance) {
            bestDistance = distance;
            bestTask = task;
        }
    }
    return bestTask;
};
```

## 去重机制

### Send 任务去重

避免重复下发同一 source→target 的发送任务：

```typescript
// 统计已排队任务
const queuedIn = Object.create(null) as Record<string, Record<string, number>>;
const queuedOut = Object.create(null) as Record<string, Record<string, number>>;

// 在计算缺口时加入排队量
const deficit = targetThreshold - (amount + pendingIn);
const surplus = amount - pendingOut;
```

### 重复检查

```typescript
const checkDuplicate = (room: string, resource: string, type: 'in' | 'out'): boolean => {
    const queued = type === 'in' ? queuedIn : queuedOut;
    return queued[room]?.[resource] > 0;
};
```

## 任务锁机制

### 锁定目的

- 防止多个 Creep 同时处理同一任务
- 避免任务重复执行

### 锁定流程

```typescript
// 1. 获取任务时锁定
const task = room.getTransportMission(creep);
if (task) {
    room.lockMissionInPool('transport', task.id, creep.id);
}

// 2. 执行任务
// ...

// 3. 提交任务时解锁
room.submitTransportMission(task.id, amount);
```

### 自动解锁

```typescript
// 检查死锁任务（绑定 Creep 死亡）
const checkDeadlock = (pool: Task[]): void => {
    for (const task of pool) {
        if (task.bindCreep && !Game.getObjectById(task.bindCreep)) {
            task.lock = false;
            task.bindCreep = null;
        }
    }
};
```

## 优缺点分析

### 优点

1. **统一管理**：所有任务集中管理，便于监控和调试
2. **优先级调度**：支持任务优先级，确保重要任务优先执行
3. **距离优化**：同优先级任务按距离分配，提高效率
4. **分帧执行**：降低 CPU 峰值，避免游戏卡顿
5. **去重机制**：避免重复任务，防止资源浪费
6. **自动清理**：自动清理已完成/失效任务，维护任务池健康

### 缺点

1. **复杂度高**：系统复杂，理解和维护成本较高
2. **内存占用**：任务池占用较多 Memory，需要定期清理
3. **调试困难**：任务分配逻辑复杂，问题定位困难
4. **锁竞争**：任务锁可能导致任务分配不均
5. **任务堆积**：任务生成速度 > 执行速度时会导致堆积

## 使用示例

### 1. 添加运输任务

```typescript
// 在房间中添加运输任务
room.TransportMissionAdd(5, {
    pos: compress(drop.pos.x, drop.pos.y),
    source: storage.id,
    target: drop.id,
    resourceType: RESOURCE_ENERGY,
    amount: 1000
});
```

### 2. 获取任务

```typescript
// Creep 获取运输任务
const task = room.getTransportMission(creep);
if (task) {
    // 执行任务
    creep.memory.transport = {
        sourceType: 'storage',
        targetId: task.data.target,
        resourceType: task.data.resourceType,
        amount: task.data.amount
    };
}
```

### 3. 提交任务

```typescript
// 完成任务后提交
const transported = creep.transfer(target, resourceType);
if (transported === OK) {
    room.submitTransportMission(task.id, amount);
}
```

### 4. 清理任务池

```typescript
// 检查并清理任务池
room.checkMissionPool('transport', (task) => {
    // 只保留 amount > 0 的任务
    return task.data.amount > 0;
});
```

## 注意事项

1. **任务ID唯一**：确保任务ID唯一，避免冲突
2. **优先级合理**：合理设置任务优先级，避免重要任务被阻塞
3. **及时清理**：定期清理已完成/失效任务，避免任务池膨胀
4. **监控任务堆积**：监控任务池大小，及时发现性能问题
5. **锁超时处理**：处理死锁任务，避免任务永久锁定
