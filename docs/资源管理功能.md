# 资源管理（ResourceManage）功能文档

---

## 概述

资源管理是一个全局资源调度和自动生产排产系统，负责跨房间资源平衡、自动生产计划分配和跨房间资源调度。该系统实现了高效的资源流动和生产管理。

---

## 核心概念

### 1. 资源阈值系统（ThresholdMap）

每个房间对每种资源定义两个阈值：
- `targetThreshold`（需求阈值）：低于此值时房间需要补充资源
- `sourceThreshold`（供应阈值）：高于此值时房间可作为供应方

```typescript
ThresholdMap[roomName][resourceType] = [targetThreshold, sourceThreshold];
```

### 2. 资源平衡（ResManageMap）

按资源维度收集可供应房间和需求房间：

```typescript
ResManageMap[resourceType] = {
    source: ['E12N34', 'E13N35'],  // 可供应房间列表
    target: ['E11N33', 'E12N33']   // 需求房间列表
};
```

### 3. 自动排产（AutoData）

为每个房间分配生产计划：

```typescript
AutoLabData[roomName][product] = limit;      // Lab 生产计划
AutoFactoryData[roomName][product] = limit;  // Factory 生产计划
```

## 资源平衡机制

### 1. 阈值判断

每轮资源管理为 eligible 房间建立资源阈值：

```typescript
// 默认来自全局配置
let [targetThreshold, sourceThreshold] = RESOURCE_BALANCE[resourceType] || [0, 0];

// 可被房间自定义配置覆盖
const custom = ResourceManage[roomName]?.[resourceType];
if (custom) {
    [targetThreshold, sourceThreshold] = custom;
}

// 判断规则
const amount = room.getResAmount(resourceType);
if (amount < targetThreshold) {
    // 房间需要补（target）
    ResManageMap[resourceType].target.push(roomName);
} else if (amount > sourceThreshold) {
    // 房间可供给（source）
    ResManageMap[resourceType].source.push(roomName);
}
```

### 2. 安全检查

```typescript
isResourceTransferInSafe() {
    if (!this.my) return false;
    if (!this.storage || !this.terminal) return false;
    if (this.level < 6) return false;

    // 基于攻防态/被破标记/冷却窗口判断
    if (this.isAttackMode) return false;
    if (this.isPowerBankBreach) return false;
    if (this.breachTimer > 0) return false;

    return true;
}
```

### 3. 成本过滤

```typescript
// 粗过滤：cost/amount > 1 时跳过
const ratio = getCostRatio(source.room.name, target.room.name);
if (ratio > 1) continue;

// 精过滤：cost > amount 时跳过
const cost = Game.market.calcTransactionCost(sendAmount, source.room.name, target.room.name);
if (cost > sendAmount) continue;
```

### 4. 批量限制

```typescript
// 避免单个目标过量调度
const perPairCap = 20e3;  // 每对房间最多发送 20k
const perSourceCap = 3;    // 每个源房间最多 3 个目标对

// 实际发送量
const desiredTotal = Math.min(
    sendAmount,
    perPairCap,
    remainingAmount
);
```

## 自动生产排产逻辑

### 1. 排产驱动流程

```
ResourceManage (全局视角)
    ↓ 计算缺口 / 构建候选列表
    ↓ 均衡分配计划（轮转策略）
AutoData (房间计划)
    ↓ 写入 AutoLabData / AutoFactoryData
    ↓ 注入原料需求阈值
ThresholdMap (资源需求)
    ↓ 跨房间调度
Send 任务 (运输任务)
```

### 2. Lab 自动排产

#### 排产逻辑

```typescript
// 1. 按 T3 → T2 → T1 优先级排序
const products = sortProductsByPriority([...T3_PRODUCTS, ...T2_PRODUCTS, ...T1_PRODUCTS]);

// 2. 计算全局缺口
for (const product of products) {
    const globalAvail = getGlobalAvail(product);
    const keep = getKeep(product);
    const def = keep > 0 ? Math.max(0, keep - globalAvail) : 0;
    if (def > 0) candidates.push({ product, def });
}

// 3. 均衡分配给各房间（轮转策略）
let cursor = 0;
for (const room of productionRooms) {
    const picked = pickCandidate(cursor);  // 轮转获取候选
    if (!picked) continue;

    // 写入计划
    AutoLabData[room.name][picked.product] = room.getResAmount(picked.product) + batch;

    // 注入原料需求
    noteNeed(room.name, raw1, inputMin);
    noteNeed(room.name, raw2, inputMin);

    cursor = (cursor + 1) % candidates.length;
}
```

#### 均衡分配策略

- **轮转分配**：房间轮流从候选列表选择任务，避免集中生产
- **最多计划数**：每房间最多 3 个计划（`maxPlansPerRoom`）
- **缺料跳过**：房间已有任务或已有未完成计划时跳过

### 3. Factory 自动排产

#### 四色链条

Factory 支持四个根资源派生的生产链：

| 根资源 | 颜色 | 派生商品 |
|--------|------|----------|
| RESOURCE_METAL | 红色 | Utrium Bar → Wire → ...
| RESOURCE_BIOMASS | 绿色 | Lemergium Bar → Cell → ...
| RESOURCE_SILICON | 蓝色 | Zynthium Bar → Alloy → ...
| RESOURCE_MIST | 黄色 | Keanium Bar → Condensate → ... |

#### 排产逻辑

```typescript
// 1. 按工厂等级分组房间
const planRoomsByLevel = {};
for (const room of productionRooms) {
    const level = room.factory.level;
    planRoomsByLevel[level] ??= [];
    planRoomsByLevel[level].push(room);
}

// 2. 构建候选商品列表
for (const product of Object.keys(COMMODITIES)) {
    const info = COMMODITIES[product];
    const level = info.level;

    // 过滤条件
    if (level < 0 || level > maxLevel) continue;       // 等级过滤
    if (!tags.colored) continue;                       // 必须是四色商品
    if (excludeWhite && tags.white) continue;           // 排除白色商品
    if ((tags.rootsMask & ~availableMask) !== 0) continue;  // 缺失根资源
    if (Object.keys(components).some(c => getGlobalAvail(c) <= 0)) continue;  // 组件不可用

    candidates.push(product);
}

// 3. 均衡分配
for (const [level, rooms] of Object.entries(planRoomsByLevel)) {
    const cursor = 0;
    for (const room of rooms) {
        const picked = pickCandidate(cursor);
        if (!picked) continue;

        // 写入计划
        AutoFactoryData[room.name][picked.product] = room.getResAmountWithFactory(picked.product) + batch;

        // 注入原料需求
        for (const [comp, need] of Object.entries(components)) {
            noteNeed(room.name, comp, need * multiplier);
        }

        cursor = (cursor + 1) % candidates.length;
    }
}
```

## 跨房间资源调度

### 1. 调度核心算法

```typescript
// 1. 源排序：按可供给余量降序
sourceRooms.sort((a, b) => getSurplus(b, res) - getSurplus(a, res));

// 2. 目标排序：按缺口降序
targetRooms.sort((a, b) => getDeficit(b, res) - getDeficit(a, res));

// 3. 匹配 source → target
for (const source of sourceRooms) {
    for (const target of targetRooms) {
        // 成本过滤
        if (getCostRatio(source, target) > 1) continue;
        const cost = Game.market.calcTransactionCost(amount, source, target);
        if (cost > amount) continue;

        // 批量优化
        const desiredTotal = Math.min(
            amount,
            perPairCap,
            terminalCapacity - currentLoad
        );

        // 下发任务
        source.room.SendMissionUpsertMax(target.room.name, resourceType, desiredTotal);
    }
}
```

### 2. 最小发送量

```typescript
const minSendAmount = {
    [RESOURCE_ENERGY]: 5000,
    'default': 1000,
    'goods': 100
};
```

### 3. 去重机制

```typescript
// 统计已排队任务
const queuedIn = Object.create(null) as Record<string, Record<string, number>>;
const queuedOut = Object.create(null) as Record<string, Record<string, number>>;

// 在计算缺口时加入排队量
const deficit = targetThreshold - (amount + pendingIn);
const surplus = amount - pendingOut;
```

## 优缺点分析

### 优点

1. **分布式决策**：每个房间独立计算资源状态，中央协调器进行全局调度
2. **渐进式收敛**：通过任务队列和阈值机制避免资源调度震荡
3. **生产-调度闭环**：生产计划驱动资源调度，资源供应支撑生产计划
4. **容错设计**：考虑终端冷却、安全状态、成本等多种边界情况
5. **灵活配置**：支持全局默认配置和房间个性化配置
6. **均衡分配**：轮转策略避免任务集中，确保各房间公平获得任务

### 缺点

1. **复杂度高**：系统复杂，理解和维护成本较高
2. **延迟收敛**：资源调度需要多轮才能收敛，响应速度较慢
3. **成本计算**：依赖 Game.market.calcTransactionCost，可能不准确
4. **死锁风险**：某些情况下可能出现资源死锁（A 等待 B，B 等待 A）
5. **内存占用**：ThresholdMap 和 ResManageMap 占用较多 Memory

## 注意事项

1. **终端冷却**：跨房间调度需要终端冷却时间，注意调度频率
2. **能量成本**：远距离调度的能量成本可能很高，合理设置阈值
3. **安全检查**：被攻击的房间不会接收资源，避免浪费
4. **库存限制**：Terminal 容量有限（300k），注意批量分配
5. **等级匹配**：Factory 等级必须与商品等级匹配，否则无法生产
