# 自动生产功能文档

---

## 概述

自动生产系统包括 autoLab（Lab 自动合成）和 autoFactory（Factory 自动生产）两个核心模块。系统根据资源缺口自动分配生产任务，并配合资源调度系统实现原料补充和产物回收。

---

## 核心概念

### 1. 生产计划（AutoData）

资源管理写入的"要生产到多少"的计划：

```typescript
AutoLabData[roomName][product] = limit;      // Lab 生产计划
AutoFactoryData[roomName][product] = limit;  // Factory 生产计划
```

### 2. 当前任务（StructData）

自动生产写入的"本房间现在做什么"：

```typescript
// Lab
StructData[roomName].labAtype / labBtype / labAmount

// Factory
StructData[roomName].factoryProduct / factoryAmount / factoryLevel
```

### 3. 搬运任务（MissionPools）

补料和清料任务：

```typescript
// Lab 补料
MissionPools[roomName].transport[]

// Factory 搬运
MissionPools[roomName].manage[]
```

### 2. 任务可行性判断

```typescript
const canStartTask = (product: ResourceConstant) => {
    const recipe = LAB_MAP[product];
    const { raw1, raw2 } = recipe;

    // 检查原料是否足够
    const labStore1 = labA.store[raw1] || 0;
    const labStore2 = labB.store[raw2] || 0;
    const storageAvail = getResAmount(raw1) + getResAmount(raw2);

    // 房间内或 Lab 内原料都 >= 1000 才能继续
    return (labStore1 >= 1000 || storageAvail >= 1000) &&
           (labStore2 >= 1000 || storageAvail >= 1000);
};
```

### 3. 缺料处理机制

#### 问题：5-999 死锁区

之前容易陷入"5-999"的死锁区：
- 房间原料 1000-9999
- Lab 内原料 0-999
- 系统判断缺料 → 清空任务
- 但实际上还能继续生产

#### 解决方案：引入维持门槛

```typescript
export const AUTO_LAB_CONFIG = {
    continueLabStoreMin: 5,          // Lab 内最小维持量
    continueInputMin: 1e3,          // 房间内最小维持量
    waitTimeoutTicks: 500,           // 缺料超时时间
};

const shouldKeepTask = () => {
    const labStoreMin = Math.min(labA.store[raw1], labB.store[raw2]);
    const storageMin = Math.min(getResAmount(raw1), getResAmount(raw2));

    // Lab 内或房间内原料都 >= 1000 则保持任务
    return labStoreMin >= 5 || storageMin >= 1000;
};
```

### 4. T1/T2/T3 任务选择

#### T1 任务（基础化合物）

```typescript
const T1_PRODUCTS = [
    RESOURCE_OH, RESOURCE_UH, RESOURCE_KO, RESOURCE_LO,
    RESOURCE_LH, RESOURCE_ZO, RESOURCE_ZH, RESOURCE_G
];

const getT1Task = () => {
    // 按 Lab 等级筛选
    const available = T1_PRODUCTS.filter(p => canProduce(p, labLevel));

    // 按缺口排序
    available.sort((a, b) => getDeficit(b) - getDeficit(a));

    // 选择第一个有缺口的
    for (const product of available) {
        if (getDeficit(product) > 0) {
            return product;
        }
    }
    return null;
};
```

## autoFactory 自动生产逻辑

### 1. 任务选择顺序

```typescript
// 1. 计划任务（优先）
if (hasAutoList) {
    task = getTask(room, getAvail, factoryLevel, autoFactoryMap, isBanned);
}

// 2. 兜底任务
if (!task) {
    const picked = getFallbackTask(room, botmem, factoryLevel, getAvail, isBanned);
    if (picked) {
        task = picked.product;
        taskAmount = picked.limit;
    }
}

// 3. 仍然没有任务则闲置
if (!task) return;
```

### 2. 计划任务选择

```typescript
const getTask = (room, getAvail, factoryLevel, autoFactoryMap, isBanned) => {
    let bestTask: ResourceConstant | null = null;
    let bestDef = 0;
    let bestLevel = -Infinity;

    for (const res in autoFactoryMap) {
        const info = COMMODITIES[res];
        const level = info.level;

        // 1. 等级兼容性检查
        if (!isFactoryLevelCompatible(factoryLevel, level)) continue;

        // 2. Ban 检查
        if (isBanned && isBanned(res)) continue;

        // 3. 限额检查
        const limit = Number(autoFactoryMap[res] || 0);
        const cur = getAvail(res);
        if (limit > 0 && cur >= limit * 0.9) continue;

        // 4. 可生产性检查
        if (!canStartTask(getAvail, res)) continue;

        // 5. 按缺口大小和等级排序
        const def = limit > 0 ? Math.max(0, limit - cur) : getCraftableOutput(getAvail, res);
        if (def > bestDef || (def === bestDef && level > bestLevel)) {
            bestDef = def;
            bestLevel = level;
            bestTask = res;
        }
    }
    return bestTask;
};
```

### 3. 等级兼容性检查

```typescript
const isFactoryLevelCompatible = (factoryLevel, commodityLevel) => {
    if (!commodityLevel) return true;  // Level 0 商品兼容所有等级
    return Number(factoryLevel) === Number(commodityLevel);
};
```

### 4. 可生产性检查

```typescript
const canStartTask = (getAvail, product) => {
    const info = COMMODITIES[product];
    const level = info.level;
    const minQuota = PRODUCTION_MIN.commodityByLevel[level] || PRODUCTION_MIN.commodityByLevel[0];
    return getCraftableOutput(getAvail, product) >= minQuota;
};

const getCraftableOutput = (getAvail, product) => {
    const info = COMMODITIES[product];
    const per = info.amount || 1;
    const components = info.components;

    let crafts = Infinity;
    for (const [c, need] of Object.entries(components)) {
        crafts = Math.min(crafts, Math.floor(getAvail(c) / need));
    }
    return crafts * per;
};
```

### 5. 缺料处理机制

#### 分类处理

```typescript
const missReasons = [];
const checkMissReasons = () => {
    for (const [comp, need] of Object.entries(components)) {
        const supplyPossible = canSupplyFromAnyRoom(room, comp);

        if (!supplyPossible) {
            missReasons.push('no_supply');      // 真正缺料
        } else if (!hasSupplyManage) {
            missReasons.push('not_scheduled'); // 未安排搬运
        }
    }
};

// 根据缺料类型采取不同策略
if (missReasons.includes('no_supply')) {
    // 真正缺料：立即清空任务
    botmem.factoryProduct = null;
} else {
    // 等待超时后再切换任务
    const waitLimit = hasAlternatives
        ? AUTO_FACTORY_CONFIG.waitTimeoutTicksWhenAlternatives  // 50 tick
        : AUTO_FACTORY_CONFIG.waitTimeoutTicks;                 // 500 tick

    if (Game.time - since > waitLimit) {
        if (fillTimeout) {
            // 填装超时：添加冷却期
            botmem.factoryBan[product] = Game.time + AUTO_FACTORY_CONFIG.banTicksAfterFillTimeout;
        }
        botmem.factoryProduct = null;
    }
}
```

### 6. 兜底策略（Fallback）

```typescript
export const AUTO_FACTORY_FALLBACK = {
    enabled: true,
    maxResolveDepth: 3,  // 递归回退深度
    zipRawSurplusMin: {   // 压缩触发底物富余量
        [RESOURCE_UTRIUM]: 20e3,
        [RESOURCE_LEMERGIUM]: 20e3,
        [RESOURCE_ZYNTHIUM]: 20e3,
        [RESOURCE_KEANIUM]: 20e3,
        [RESOURCE_GHODIUM]: 20e3,
        [RESOURCE_OXYGEN]: 20e3,
        [RESOURCE_HYDROGEN]: 20e3,
        [RESOURCE_CATALYST]: 20e3,
    },
    keepInRoom: {         // 目标库存上限
        [RESOURCE_COMPOSITE]: 3e3,
        [RESOURCE_CRYSTAL]: 3e3,
        [RESOURCE_LIQUID]: 3e3,
        // ...
    },
    keepByLevelInRoom: {  // 按等级的兜底保有量
        0: 5e3,
        1: 2e3,
        2: 1e3,
        3: 500,
        4: 200,
        5: 50,
    },
};
```

#### 兜底任务优先级

```typescript
const getFallbackTask = () => {
    // 1) 压缩：zipMap 全量（不局限 mineralType）
    for (const [raw, zip] of Object.entries(zipMap)) {
        const rawKeep = zipRawSurplusMin[raw];
        if (rawKeep && getAvail(raw) < rawKeep) continue;
        const picked = pickKeeped(String(zip));
        if (picked) return picked;
    }

    // 2) 白色根商品
    for (const product of [RESOURCE_COMPOSITE, RESOURCE_CRYSTAL, RESOURCE_LIQUID]) {
        const picked = pickKeeped(product);
        if (picked) return picked;
    }

    // 3) 基础四色中间件
    for (const product of [RESOURCE_WIRE, RESOURCE_CELL, RESOURCE_ALLOY, RESOURCE_CONDENSATE]) {
        const picked = pickKeeped(product);
        if (picked) return picked;
    }

    // 4) 四色链条兜底：按工厂有效等级，挑"最缺的可生产项"
    const keep = keepByLevelInRoom[effectiveLevel] || 0;
    for (const product of Object.keys(COMMODITIES)) {
        const info = COMMODITIES[product];
        const level = info.level;
        if (level !== effectiveLevel) continue;

        const have = getAvail(product);
        if (have >= keep) continue;

        const resolved = resolveProducible(product, 0, getMinQuota(product));
        if (resolved) return { product: resolved.product, limit: resolved.limit };
    }

    // 5) 降级策略：当所有任务都达到目标库存时，降低阈值重新尝试
    const degradeRatio = 0.5;
    let degradeAttempt = 0;
    while (degradeAttempt < 3) {
        for (const product of [...WHITE_ROOTS, ...BASIC_INTERMEDIATES]) {
            const keep = keepInRoom[product] || 0;
            const reducedKeep = keep * Math.pow(degradeRatio, degradeAttempt);
            if (getAvail(product) >= keep && getAvail(product) < reducedKeep) {
                const resolved = resolveProducible(product, 0, getMinQuota(product));
                if (resolved) return { product: resolved.product, limit: reducedKeep };
            }
        }
        degradeAttempt++;
    }

    return null;
};
```

#### 分配逻辑

```typescript
const maxPlans = maxPlansPerRoom ?? 1;
const assignedProducts = new Set<string>();

for (let planIdx = 0; planIdx < maxPlans; planIdx++) {
    const picked = pickForRoom();
    if (!picked) break;

    const def = picked.def;
    const isPrimary = planIdx === 0;
    // 主任务使用 batchPerRoom，备用任务使用 batchPerRoom / 2
    const batch = Math.max(0, Math.min(
        RESOURCE_PRODUCTION.factory.chain.batchPerRoom / (isPrimary ? 1 : 2),
        def
    ));

    const minBatch = PRODUCTION_MIN.commodityByLevel[level] || PRODUCTION_MIN.commodityByLevel[0];
    if (batch < minBatch) continue;

    remaining[`${level}:${picked.product}`] = def - batch;
    autoFactoryMap[picked.product] = getRoomAvailWithFactory(room, picked.product) + batch;
    assignedProducts.add(picked.product);

    tryLog([
        `FACTORY ${room.name}`,
        `产物: ${picked.product}`,
        `缺口: ${def}`,
        `批量: ${batch}`,
        `类型: ${isPrimary ? '主' : '备用'}`
    ]);
}
```

## Lab/Factory 控制逻辑

### LabWork

```typescript
const LabWork = () => {
    const product = StructData.labAtype;
    if (!product) return;

    const labA = getLab(product, 'A');
    const labB = getLab(product, 'B');

    if (!labA || !labB) return;

    const components = LAB_MAP[product];
    const { raw1, raw2 } = components;

    // 检查原料是否足够
    if (labA.store[raw1] < LAB_MIN_AMOUNT || labB.store[raw2] < LAB_MIN_AMOUNT) {
        return;
    }

    // 执行反应
    for (const lab of labs) {
        if (lab.id === labA.id || lab.id === labB.id) continue;
        const result = lab.runReaction(labA, labB);
        if (result !== OK && result !== ERR_TIRED) {
            console.log(`Lab reaction failed: ${result}`);
        }
    }
};
```

### FactoryWork

```typescript
const FactoryWork = () => {
    const factory = room.factory;
    if (!factory) return;

    const product = StructData.factoryProduct;
    if (!product) return;

    const level = Number(factory.level || StructData.factoryLevel || 0);
    const info = COMMODITIES[product];
    if (info && info.level && Number(info.level) !== level) {
        console.log(`Factory level mismatch: ${level} != ${info.level}`);
        return;
    }

    const components = info?.components || {};

    // 检查原料是否足够
    for (const [comp, need] of Object.entries(components)) {
        if (factory.store[comp] < Number(need)) {
            return;  // 原料不足，等待补料
        }
    }

    // 执行生产
    const result = factory.produce(product);
    if (result !== OK && result !== ERR_NOT_ENOUGH_RESOURCES && result !== ERR_TIRED) {
        console.log(`Factory production failed: ${result}`);
    }
};
```

```typescript
const UpdateLabTransportMission = () => {
    const product = StructData.labAtype;
    if (!product) return;

    const components = LAB_MAP[product];
    const { raw1, raw2 } = components;

    const labA = getLab(product, 'A');
    const labB = getLab(product, 'B');

    // 补 raw1
    const need1 = LAB_MIN_AMOUNT - (labA?.store[raw1] || 0);
    if (need1 > 0) {
        room.TransportMissionAdd(level, {
            pos: compress(labA.pos.x, labA.pos.y),
            source: storage.id,
            target: labA.id,
            resourceType: raw1,
            amount: need1
        });
    }

    // 补 raw2
    const need2 = LAB_MIN_AMOUNT - (labB?.store[raw2] || 0);
    if (need2 > 0) {
        room.TransportMissionAdd(level, {
            pos: compress(labB.pos.x, labB.pos.y),
            source: storage.id,
            target: labB.id,
            resourceType: raw2,
            amount: need2
        });
    }

    // 清 Lab
    const cleanAmount = FACTORY_MAX;
    for (const lab of labs) {
        for (const res in lab.store) {
            if (res !== raw1 && res !== raw2 && res !== product) {
                room.TransportMissionAdd(level, {
                    pos: compress(lab.pos.x, lab.pos.y),
                    source: lab.id,
                    target: storage.id,
                    resourceType: res,
                    amount: lab.store[res]
                });
            }
        }
    }
};
```

### Factory 搬运任务

```typescript
const CheckFactoryResAmount = (room) => {
    const factory = room.factory;
    const mem = StructData[room.name];
    const product = mem.factoryProduct;

    // 1. 关停时全部取出
    if (!mem.factory || !product) {
        for (const type in factory.store) {
            room.ManageMissionAdd('f', 's', type, factory.store[type]);
        }
        return;
    }

    const components = COMMODITIES[product]?.components || {};

    // 2. 清理非材料非产物
    for (const type in factory.store) {
        if (!components[type] && type !== product) {
            room.ManageMissionAdd('f', 's', type, factory.store[type]);
        }
    }

    // 3. 补充原料
    for (const component in components) {
        const amount = FACTORY_MIN - factory.store[component];
        room.ManageMissionAdd('s', 'f', component, amount);
        if (storage.store[component] < amount) {
            room.ManageMissionAdd('t', 'f', component, amount - storage.store[component]);
        }
    }

    // 4. 清理产物
    if (factory.store[product] >= FACTORY_MAX) {
        room.ManageMissionAdd('f', 's', product, FACTORY_MAX);
    }
};
```

## 优缺点分析

### 优点

1. **层次化任务选择**：从自定义到 T1/T2/T3，逐级降级
2. **智能缺料处理**：区分不同场景，避免资源浪费
3. **完善的兜底机制**：确保生产设施不会闲置
4. **灵活的配置选项**：支持房间级别和全局级别控制
5. **超时去抖机制**：避免任务频繁切换造成性能问题
6. **多任务分配**：备用任务机制提高生产连续性
7. **降级策略**：库存达标后仍能生产储备

### 缺点

1. **复杂度高**：系统复杂，理解和维护成本较高
2. **依赖资源调度**：需要稳定的资源调度支持
3. **调试困难**：任务选择逻辑复杂，问题定位困难
4. **Ban 冷却**：填装超时后 200 tick 冷冻期可能导致闲置
5. **轮转阻塞**：之前的轮转分配问题可能导致部分房间闲置（已修复）

## 注意事项

1. **Lab 等级**：不同 Lab 等级支持不同的 T1/T2/T3 产物
2. **Factory 等级**：Factory 等级必须与商品等级匹配
3. **原料供应**：确保资源调度系统正常工作，原料能及时补充
4. **Ban 冷却**：填装超时后商品会被 Ban 200 tick，期间无法选择
5. **最小生产额度**：生产必须达到最小生产额度，否则任务会被跳过
6. **Boost 消耗**：生产 T2/T3 产物会消耗 Boost，注意 Lab Boost 预约
